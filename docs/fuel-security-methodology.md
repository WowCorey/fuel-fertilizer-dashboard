# Fuel Security Dashboard Methodology

Last reviewed: 2026-04-23

This document defines the first public-source fuel-security dashboard surface.
It is intentionally conservative: values are shown only when they are observed
from a named public source or derived mechanically from an existing verified
envelope.

## Source Priority

Priority 1 sources are official Australian public sources already aligned with
the repository's evidence model:

- PM&C public information on fuel supply, compiled by DCCEEW.
- DCCEEW Australian Petroleum Statistics.
- DCCEEW Fuel Security Services Payment and stockholding disclosures.
- ABS and state public fuel-price feeds already registered in `data/sources.yml`.

No paywalled, scraped or non-reproducible source is used for the fuel-security
page.

## Status Logic

The dashboard currently shows two separate concepts:

- `Official public level`: the observed PM&C National Fuel Security Plan level.
- `Dashboard status model`: unavailable.

The dashboard does not map PM&C Level 1/2/etc. into its own Stable, Tight,
Disrupted or Critical labels. That would be a new interpretation and is not yet
supported by enough public operational coverage.

The dashboard status model may only be enabled when all of these are true:

- PM&C/DCCEEW national plan level is loaded and fresh.
- Product-level days, reserves and stock context are loaded.
- Outage/stock-out coverage is sufficient and its geographic limits are visible.
- Import/order/tanker context is loaded.
- Terminal/storage visibility limits are explicitly handled.
- Every status label has a documented rule and test.

Until then, the model fails closed and displays `Status unavailable`.

## Days Remaining

The first product days envelopes are:

- `fuel_security_petrol_days_remaining`
- `fuel_security_diesel_days_remaining`
- `fuel_security_jet_days_remaining`

These are generated by selecting product-specific fields from
`pmc_mso_days_cover.extra.fields`:

| Output envelope | Parent field |
|---|---|
| `fuel_security_petrol_days_remaining` | `petrol_days` |
| `fuel_security_diesel_days_remaining` | `diesel_days` |
| `fuel_security_jet_days_remaining` | `jet_fuel_days` |

This is labelled `Derived` because the page reshapes one verified PM&C table
into product-specific dashboard envelopes. It is not an independent burn-rate
calculation. The dashboard must not present these values as terminal-by-terminal
stock cover or total national IEA compliance.

## Product Supply Position

The product table combines:

- PM&C MSO reserve volumes from `pmc_mso_fuel_reserves`.
- PM&C product days from the derived days envelopes.
- APS product stocks where the APS workbook exposes a product stock series.
- APS product sales as monthly demand context.
- APS product imports where a product import series is loaded.

Blank cells remain blank when the source is not loaded. Jet fuel currently uses
PM&C reserve/days and APS sales context, but no loaded APS jet-import or
jet-stock envelope is shown on the fuel-security page.

## Outage And Disruption Visibility

PM&C publishes state/territory retail stock-out counts as a dated public
snapshot. The current table does not publish an Australia-wide petrol total.
That absence must remain visible.

The dashboard does not currently load:

- a live national dry-station feed,
- station-level outage records,
- inferred regional shortages from price spikes.

The placeholder envelope `fuel_security_live_station_outage_feed` records this
gap so the page can label it `Unavailable`.

## Import And Shipping Risk

The page uses:

- PM&C forward import-order volume for the next four weeks.
- PM&C aggregate ships-on-water counts.
- ABS petroleum import value.
- APS product import context.

PM&C states that ships-on-water counts are estimated by DCCEEW using Kpler
commodity terminal data. The repository does not load Kpler directly and does
not publish vessel identities, AIS positions or ETAs.

The placeholder envelope `fuel_security_live_vessel_tracking` records this gap.

## Storage And Terminal Visibility

The page uses:

- PM&C MSO reserves by product.
- APS national/product stock series where loaded.

The dashboard does not publish terminal-by-terminal capacity, terminal outage
status or private inventory telemetry. The placeholder envelope
`fuel_security_terminal_capacity` records this gap.

## Trust Labels

The fuel-security page uses these labels:

| Label | Meaning |
|---|---|
| Observed | Copied or fetched from a named public source. |
| Derived | Mechanically calculated or selected from verified envelope data. |
| Manual | Hand-keyed from a named public source. |
| Stale | Loaded but outside the source cadence freshness window. |
| Partial coverage | Public source covers only part of the desired operational view. |
| Unavailable | No defensible public source or method is loaded. |
| Scenario | Reserved for future what-if calculations. |
| Estimated | Reserved; do not use unless the estimate method and source support are explicit. |

No hidden estimate should appear without one of these labels.
